# Architecture

MiniPACS is a single-host, docker-compose-orchestrated PACS portal for
independent clinics. One machine in the clinic owns every layer ‚Äî DICOM
ingest, image storage, indexing, viewer, auth, audit, and public web
access ‚Äî with no cloud dependency beyond Cloudflare Tunnel for TLS
termination on WAN traffic.

## Request paths

### WAN client (clinician at home, patient opening a share link)

```
Browser
  ‚îÇ HTTPS (TLS terminated at CF edge)
  ‚ñº
Cloudflare edge
  ‚îÇ CF Tunnel (outbound-only, no inbound ports)
  ‚ñº
cloudflared container
  ‚îÇ HTTP, docker network
  ‚ñº
frontend (nginx)
  ‚îÇ static SPA served, /api/* reverse-proxied
  ‚ñº
backend (FastAPI)
  ‚îÇ SQL over asyncpg pool, cache over redis
  ‚ñº
postgres + redis  ‚îÄ‚îÄ‚ñ∫  orthanc  ‚îÄ‚îÄ‚ñ∫  /var/lib/orthanc/db (DICOM files)
```

### LAN client (radiologist on the clinic Wi-Fi)

```
Browser
  ‚îÇ HTTPS (TLS terminated at the clinic box, HTTP/2)
  ‚îÇ DNS: pacs.your-clinic.example ‚Üí 10.0.0.50  (UniFi split-horizon override)
  ‚ñº
Windows host :443  (netsh portproxy ‚Üí WSL)
  ‚ñº
frontend (nginx listens on 443 with CF Origin Certificate)
  ‚ñº  ... same rest of the chain as WAN
```

Typical LAN request skips ~50‚Äì100 ms of CF round-trip and muxes Stone's
DICOMweb bursts over a single HTTP/2 connection.

### Modality (MRI, CT, X-ray, US)

```
Modality console
  ‚îÇ DICOM C-STORE / C-ECHO
  ‚ñº
Windows host :48924  (netsh portproxy ‚Üí WSL)
  ‚ñº
orthanc :4242   (DICOM protocol, not HTTP)
```

### Live worklist event


## Services

| Container | Image | Role |
|---|---|---|
| `postgres` | `postgres:16-alpine` | Shared index ‚Äî Orthanc schema AND MiniPACS backend tables (users, shares, audit, settings‚Ä¶). Disjoint table names, one `pg_dump` covers both. |
| `orthanc` | `orthancteam/orthanc` | DICOM C-STORE/C-ECHO server + DICOMweb + Stone Web Viewer plugin + OHIF plugin + Python plugin for thumbs & WS notifications. PG-indexed. |
| `redis` | `redis:7-alpine` | QIDO cache (find_studies/find_patients). No persistence; 128 MB allkeys-lru. Fallback-friendly ‚Äî backend degrades to in-memory if Redis disappears. |
| `backend` | python:3.12-slim + FastAPI | REST + WebSocket + audit. asyncpg pool. Serves `/api/*`. Aggregate endpoints `/api/boot`, `/api/dashboard`, `/api/studies/{id}/full`. |
| `frontend` | node build ‚Üí nginx:alpine | React SPA (Vite, code-split, PWA via vite-plugin-pwa). nginx reverse-proxies `/api/*`, `/dicom-web/*`, `/orthanc/*`, `/ohif/*`, `/stone-webviewer/*`. |
| `cloudflared` | `cloudflare/cloudflared` | Outbound tunnel to Cloudflare edge ‚Äî zero inbound firewall rules needed for WAN access. |

## Storage layout

| Volume | Path in container | Content |
|---|---|---|
| `orthanc-data` | `/var/lib/orthanc/db` | DICOM files on disk (pixel data). Orthanc's `StorageDirectory`. |
| `orthanc-pg` | `/var/lib/postgresql/data` | Postgres data dir ‚Äî both Orthanc's index AND MiniPACS backend tables. |
| `minipacs-db` | `/app/data` | Legacy SQLite ‚Äî kept only for migration from pre-PG installs; no longer written. |
| `minipacs-thumbs` | `/var/cache/minipacs-thumbs` | PNG thumbs pre-generated by Orthanc Python plugin, shared read-only with backend. |
| `nginx-cache` | `/var/cache/nginx` | Disk cache for DICOMweb metadata + WADO frames (7-day TTL, 2 GB cap). |
| `./ssl` | `/etc/ssl/minipacs:ro` | Bind mount for the Cloudflare Origin Certificate on the LAN HTTPS listener. Self-signed fallback baked into the image. |

## Caching hierarchy

Every read-heavy path has multiple cache layers so the slow-disk
fallback is the exception, not the rule:

```
browser
  ‚îú‚îÄ React Query cache (per-tab, 5-min GC)
  ‚îî‚îÄ Service Worker (Workbox: NetworkFirst /api/, CacheFirst /stone-webviewer/)
      ‚îÇ
      ‚ñº
    nginx
      ‚îú‚îÄ DICOMweb metadata + WADO frames on disk (2 GB, 7-day TTL)
      ‚îî‚îÄ /api/ CORS preflight (24h)
          ‚îÇ
          ‚ñº
        backend
          ‚îú‚îÄ Redis QIDO cache (30s fresh / 600s stale envelope)
          ‚îî‚îÄ in-process httpx pool to Orthanc (100/50, long keepalive)
              ‚îÇ
              ‚ñº
            orthanc
              ‚îú‚îÄ SQLite page cache 1.5 GB (mmap 2 GB)
              ‚îú‚îÄ DICOMweb SeriesMetadataCacheSize 30000 entries
              ‚îî‚îÄ ExtraMainDicomTags ‚Äî tags resolved from index, not disk
                  ‚îÇ
                  ‚ñº
                disk (DICOM files / PG data dir)
```

## Schema boundary between backend and Orthanc

Both applications connect as PG user `orthanc` to database `orthanc`,
but their tables never overlap:

| Owner | Tables |
|---|---|
| Orthanc | `resources`, `attachedfiles`, `metadata`, `mainpatienttags`, `mainstudytags`, `mainseriestags`, `mainidentifiertags`, `globalintegers`, `globalproperties`, `changes`, `deletedfiles`, `deletedresources`, `patientrecyclingorder`, `remainingancestor`, `serverproperties`, `maincomputedresources`, `maincomputedtags` |
| MiniPACS backend | `users`, `patient_shares`, `pacs_nodes`, `transfer_log`, `audit_log`, `external_viewers`, `settings`, `study_reports` |

A single `pg_dump orthanc` captures everything ‚Äî one backup artefact
per tick instead of two.

## Authentication surfaces

| Surface | Mechanism |
|---|---|
| `/api/*` (except `/api/ws/*` and `/api/internal/*`) | JWT bearer, HS256, 30-min access + 7-day refresh. `token_version` per user ‚Äî bumping it (e.g. on password change) invalidates all outstanding tokens for that user. |
| `/api/ws/studies` | Same JWT, passed via `?token=‚Ä¶` query string (browsers cannot set `Authorization` header on WebSocket open). Policy-violation close on invalid token. |
| `/api/internal/events/*` | Shared-secret `X-Internal-Token` header + RFC1918-only client IP check. Rotates with `scripts/rotate-secrets.sh`. |
| Patient portal `/p/{token}` | Unique per-share token + optional PIN (bcrypt-hashed). httponly cookie proves PIN satisfaction for the session. |
| Orthanc `/dicom-web/*`, `/orthanc/*`, `/ohif/*`, `/stone-webviewer/*` | nginx injects HTTP Basic `${ORTHANC_BASIC_AUTH}` on the way to Orthanc ‚Äî frontend never sees Orthanc credentials. |

## Durability & recovery

- **Backup cron** (`scripts/backup.sh`, 02:00 daily):
  - `pg_dump -Fc orthanc > pg-YYYYMMDD.dump` ‚Äî covers everything in
    both applications' schemas.
  - `tar czf orthanc-YYYYMMDD.tar.gz /var/lib/orthanc/db` ‚Äî DICOM files.
  - 30-day retention in `backups/`.
- **Off-site target** still TODO (HIPAA 3-2-1). Planned: rclone to
  Backblaze B2.
- **WSL autostart** (`MiniPACS_Boot` Scheduled Task + systemd unit
  inside WSL): if the Windows host reboots or WSL crashes, the stack
  is back without human intervention in ‚â§60s.
- **Liveness watchdog** (`MiniPACS_Watchdog`): every 5 min probes
  `/api/health`; after 2 consecutive failures re-fires the boot
  sequence. Two-tick threshold prevents cascading restarts during
  normal container-recreate windows.

See [docs/wsl-autostart.md](wsl-autostart.md) and [docs/prod-hardening.md](prod-hardening.md)
for operational details.

## Viewer strategy

Two viewers ship, both from the Orthanc team:

- **Stone Web Viewer** ‚Äî default, ~1 MB WASM, cold-open <1 s on single-
  study URLs. Uses QIDO/WADO on demand; does not need the OHIF dicom-
  json attachment. Skips SR/OT/KO/DOC/SEG/PR series which Stone's
  `DicomResourcesLoader` cannot render.
- **OHIF Viewer** ‚Äî secondary, hosted by Orthanc's OHIF plugin at
  `/ohif/`. Used as fallback and for workflows Stone does not cover.
  Consumes a dicom-json attachment precomputed by Orthanc when each
  study becomes STABLE ‚Äî zero disk I/O at viewer open.

Viewer choice is per-study-link: the backend's `external_viewers` table
provides the URL template the UI renders. Admins can enable other
viewers (OsiriX, RadiAnt, 3D Slicer, MicroDicom, PostDICOM, MedDream)
from Settings ‚Äî those hand off via `osirix://`/`radiant://` URL schemes
or upload to a cloud counterpart.

## Progressive Web App

`frontend` builds as a PWA via `vite-plugin-pwa` with Workbox:

- Installable (Add to Home Screen / Dock).
- Offline-capable shell ‚Äî last-viewed worklist renders without
  network; opens studies throw a clear "offline" toast rather than a
  blank screen.
- Service worker strategies:
  - `NetworkFirst` for `/api/*` (fresh data when online, stale when
    offline).
  - `CacheFirst` for `/stone-webviewer/*` (WASM rarely changes, cache
    aggressively).
  - `NetworkOnly` for `/dicom-web/*` (image data is not cached by
    SW ‚Äî that's nginx's job, not the browser's).

## Non-goals

- **Multi-tenant hosting.** One MiniPACS deployment serves one clinic.
  Two clinics ‚Üí two deployments (both free). We don't namespace the
  data layer to fake multi-tenancy.
- **Reading reports / primary diagnosis.** Stone has "Not For
  Diagnostic Use" disabled (UI branding), but the architecture assumes
  that diagnostic reading happens elsewhere. MiniPACS is the clinic's
  storage + sharing + viewing portal, not a replacement for a RIS.
- **Cloud scale.** Designed for a single NVMe SSD, 16 GB RAM, one host.
  Rack-scale PACS is a different product.
